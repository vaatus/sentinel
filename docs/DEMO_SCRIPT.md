# Demo video script — Sentinel

**Length:** 2:00 · **Format:** screen recording + voice-over · **Tone:** calm, enterprise, evidence-driven.

> The pitch: *Sentinel is compliance-as-CI for healthcare and fintech. Built with IBM Bob IDE — twelve sessions, $18.72 of Bobcoin, every shipped feature traceable to a task ID. The product blocks non-compliant code at the PR gate and verifies its own fixes.*

---

## Pre-recording checklist

- [ ] Terminal font ≥ 18pt, dark theme, monospace.
- [ ] Run once to warm the cache: `npm install && npm run build`.
- [ ] Clean state: `rm -rf demo-clinic-app/.sentinel/findings.sqlite demo-clinic-app/.sentinel/findings.json`.
- [ ] Open Bob IDE in a second window with `.bob/custom_modes.yaml` visible (for the 0:40 cutaway).
- [ ] Open the dashboard at `localhost:3000` in a browser tab (warm it).
- [ ] Have a fresh clone of any GitHub repo open in a third terminal (for the `init` scene).

---

## Storyboard

### 0:00 – 0:10 · Title + hook

- **On-screen:** Title card → "Sentinel — Compliance-as-CI for healthcare and fintech. Built with IBM Bob."
- **VO:** *"Healthcare and fintech spend up to fifty million dollars a year on compliance audits. Sentinel — built with IBM Bob IDE — moves that work from a six-month cycle into the four minutes between push and merge."*

### 0:10 – 0:25 · One-command install (the strongest hook scene)

- **On-screen:** A fresh repo in a terminal. Type:
  ```
  npx sentinel init
  ```
- **Show:** the interactive wizard — multiselect framework prompt, blocker-level select, workflow confirm. Hit enter through defaults. Watch `.bob/`, `.github/workflows/`, `.sentinel/` appear in the file tree.
- **VO:** *"Ten seconds to install. Every developer's editor gets the five Bob IDE compliance modes. Every pull request gets gated. Every PR comment is generated by Bob."*

### 0:25 – 0:45 · CLI scan — show the breadth

- **On-screen:** Terminal in `demo-clinic-app/`. Run **all four** framework scans in sequence:
  ```
  sentinel scan --framework hipaa --path ./demo-clinic-app
  sentinel scan --framework soc2  --path ./demo-clinic-app
  sentinel scan --framework pci   --path ./demo-clinic-app
  sentinel scan --framework gdpr  --path ./demo-clinic-app
  ```
- **Show:** the four summary boxes. Highlight: HIPAA 4/100 (very broken), SOC2 76/100, PCI 68/100, GDPR 76/100.
- **VO:** *"Thirty controls across four frameworks — HIPAA, SOC 2, PCI, GDPR. Same codebase, four lenses. Bob designed every control."*

### 0:45 – 0:55 · Bob IDE cutaway (10 seconds, mandatory eligibility shot)

- **On-screen:** Flip to Bob IDE window. Open `.bob/custom_modes.yaml`. Briefly highlight `hipaa-auditor` mode → trigger `/audit-hipaa` in the chat → freeze on Bob producing structured findings inline.
- **VO:** *"This is the Bob IDE skill pack — drop it into any team's editor and every keystroke is auditor-aware. Same prompt templates as the CLI."*

### 0:55 – 1:15 · Dashboard + PHI flow

- **On-screen:** `sentinel dashboard`. Show:
  - Overview card with red 4/100 ring + severity breakdown
  - Findings table — click a row → file:line + Bob's reasoning + remediation hint
  - Open `/data-flow` → pan across PHI source → log sink graph
- **VO:** *"This is the PHI data flow Bob mapped across the codebase. Patient data flowing into a logger — Section 164.312(b). The kind of finding a regex can't catch and an auditor takes a week to find."*

### 1:15 – 1:40 · Auto-remediate + **closed-loop verification** (the differentiator)

- **On-screen:** Click **Auto-remediate with Bob**. Watch 7 patches generate. Click into one PR — show the diff + blast-radius panel.
- **Then:** scroll to the **verification badges** on each PR row. Some `verified-resolved` (green), some `regression` (red) — the green ones show "fix confirmed, finding gone." Red ones show "patch didn't actually resolve the violation."
- **VO:** *"Here is what no other compliance scanner does. After Bob generates a patch, Sentinel re-audits the patched code and labels every fix — verified-resolved, or regression. The agent verifies its own work. Bad fixes never reach a human reviewer."*

### 1:40 – 1:55 · PR gate — block in real time

- **On-screen:** Switch to GitHub. Show a PR with the sticky **Sentinel compliance report** comment + a red failing check. Highlight the "❌ This PR is blocked" banner.
- **VO:** *"Drop the workflow into any healthcare or fintech repo and non-compliant code stops merging. Fifty-thousand-dollar audits become a zero-cost CI gate."*

### 1:55 – 2:00 · Close

- **On-screen:** Three-line card:
  - *$50K external audit + 6 months → $0 + 4 minutes per PR*
  - *Built with IBM Bob IDE — 12 sessions, $18.72 of Bobcoin*
  - *github.com/vaatus/sentinel*
- **VO:** *"Sentinel. Built with IBM Bob."*

## Recording tips

- **Don't talk over fast terminal output.** Pause the VO when a spinner is running and let the user *see* the output.
- **Re-record any scene where you stumble** — there's only 120 seconds, every word has to land.
- **Watch the audio levels** — keep VO at around -12 dB, no music underneath (compliance pitch, not a hype reel).
- **Pre-render the install scene** — if `npx sentinel init` takes 8 seconds and you only have 15 for the scene, edit out dead time.
- **Optional A-roll if you have time:** at the very end, a 2-second clip of the `bob_sessions/` folder open in Finder showing 12 paired `.md` + `.png` files. Visual proof of the Bob receipts.
